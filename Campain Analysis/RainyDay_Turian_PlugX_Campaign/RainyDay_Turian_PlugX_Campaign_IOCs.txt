## Attack Techniques

### DLL Search Order Hijacking
- **Technique**: T1574.001 - DLL Search Order Hijacking
- **Description**: Abuse of Windows DLL search order to load malicious DLLs
- **Implementation**: Placing malicious DLLs in directories searched before legitimate ones
- **Evasion**: Bypassing security controls through legitimate-looking DLL loading

### Masquerading
- **Technique**: T1036 - Masquerading
- **Description**: Hiding malicious activities behind legitimate-looking processes
- **Implementation**: Using legitimate process names and file locations
- **Evasion**: Avoiding detection through process masquerading

### Debugger Evasion
- **Technique**: T1622 - Debugger Evasion
- **Description**: Detecting and avoiding debugger environments
- **Implementation**: Anti-debugging techniques and environment checks
- **Evasion**: Preventing analysis and reverse engineering

### Process Injection
- **Technique**: T1055 - Process Injection
- **Description**: Injecting malicious code into legitimate processes
- **Implementation**: DLL injection and process hollowing
- **Evasion**: Hiding malicious activities within legitimate processes

## Technical Details

### DLL Search Order Hijacking
- **Windows DLL Search Order**: 
  1. Application directory
  2. System directory (C:\Windows\System32)
  3. 16-bit system directory (C:\Windows\System)
  4. Windows directory (C:\Windows)
  5. Current directory
  6. Directories in PATH environment variable
- **Abuse Vector**: Placing malicious DLLs in directories searched before legitimate ones
- **Evasion**: Using legitimate-looking DLL names and locations

### Evasion Techniques
- **Anti-Debugging**: Detecting debugger presence and altering behavior
- **Anti-Analysis**: Detecting analysis environments and evading detection
- **Process Masquerading**: Hiding behind legitimate processes
- **File Masquerading**: Using legitimate file names and locations

### Persistence Mechanisms
- **DLL Hijacking**: Persistent DLL loading through search order abuse
- **Process Injection**: Long-term persistence through process injection
- **Registry Modification**: Registry-based persistence mechanisms
- **Service Installation**: Service-based persistence
